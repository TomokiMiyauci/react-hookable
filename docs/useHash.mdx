# useHash

Tracks location hash value.
Due to `hitory.replaceState`, no history is kept.

## Usage

```tsx
import { useHash } from 'react-hookable'

const [hash, setHash] = useHash()

setHash('searching')
// URL: http*/#searching
// hash: #searching
```

### initialState

```tsx
import { useHash } from 'react-hookable'

const [hash, setHash] = useHash('calling')
// URL: http*/#calling
// hash: #calling
```

### options

#### hashMark

```tsx
import { useHash } from 'react-hookable'

const [hash, setHash] = useHash(undefined, { hashMark: false })

setHash('searching')
// URL: http*/#searching
// hash: searching
```

## Advanced

Usually, if you rewrite a hash with `hitory.replaceState`, you need to be a bit creative to remove the # sign.

In the following example, only the `#` sign remains in the URL. (There's no problem in this, other than the appearance).

```tsx
// URL: http*/#searching
window.history.replaceState(null, document.title, '') // or `undefined`
// URL: http*/#
```

`useHash` solves this problem. With `useHash`, empty string is a bit special.
If you update with an empty string, the `#` sign will disappear cleanly.

```tsx
import { useHash } from 'react-hookable'

const [hash, setHash] = useHash()
// URL: http*/#searching
// hash: #searching

setHash('')
// URL: http*/
// hash: ''
```
