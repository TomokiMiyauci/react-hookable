# useHashState

Tracks location hash value.
Due to `hitory.replaceState`, no history is kept.

## Usage

```tsx
import { useHashState } from 'react-hookable'

const [hash, setHash] = useHashState()

setHash('searching')
// URL: http*/#searching
// hash: #searching
```

### initialState

```tsx
import { useHashState } from 'react-hookable'

const [hash, setHash] = useHashState('calling')
// URL: http*/#calling
// hash: #calling
```

### options

#### hashMark

```tsx
import { useHashState } from 'react-hookable'

const [hash, setHash] = useHashState(undefined, { hashMark: false })

setHash('searching')
// URL: http*/#searching
// hash: searching
```

## Advanced

Usually, if you rewrite a hash with `hitory.replaceState`, you need to be a bit creative to remove the # sign.

In the following example, only the `#` sign remains in the URL. (There's no problem in this, other than the appearance).

```tsx
// URL: http*/#searching
window.history.replaceState(null, document.title, '') // or `undefined`
// URL: http*/#
```

`useHashState` solves this problem. With `useHashState`, empty string is a bit special.
If you update with an empty string, the `#` sign will disappear cleanly.

```tsx
import { useHashState } from 'react-hookable'

const [hash, setHash] = useHashState()
// URL: http*/#searching
// hash: #searching

setHash('')
// URL: http*/
// hash: ''
```
